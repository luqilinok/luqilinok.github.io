C++之继承

Posted on 2019-07-08   |   更新于 2019-07-08 | In Notes

面向对象的三大特性：封装、继承和多态，总结一下C++的继承以及注意事项。

继承关系

C++共有3种继承方式，分别是：public、protected和private，一般语法为：

class 类名：[继承方式] 基类名

{

};

如果不写继承方式，默认是private继承，继承方式和访问修饰符的关系如下：

继承方式 public成员 protected成员 private成员
public public protected 不可见
protected protected protected 不可见
private private private 不可见

注意：

1.使用using关键字能够改变基类成员再派生类中的访问权限，例如将public改为private，将protected改为public，但是不能改变private成员的访问权限；

2.同名隐藏：子类和父类中有同名成员时，子类成员将屏蔽父类堆成员的直接访问。（在子类成员函数中，可以使用 基类::基类成员 访问父类成员），主要名字相同时就会发生隐藏，这个应该避免；

3.函数覆盖：子类对象和父类对象的函数名字和参数相同时，就会发生函数覆盖；

4.函数重载：函数名字相同，参数列表不同（参数类型、参数个数），就会发生函数重载；

5.构造函数、析构函数、拷贝构造函数、拷贝赋值运算符不能被继承；

6.继承中构造函数的调用顺序：基类构造函数->派生类中对象构造函数->派生类构造函数，析构函数调用顺序刚好相反。

多继承和菱形继承

一个子类只有一个直接父类，这种关系为单继承。
一个子类两个或以上的直接父类，这种关系为多继承，多继承里面比较典型的菱形继承。

菱形继承会产生二义性和数据冗余的问题，解决这个的办法是使用虚继承。

友元函数和static继承

1.友元函数不能被子类继承，派生类的friend函数可以访问派生类本身的一切变量，包括父类继承下来的protected域中的变量。但是对于父类来说，它并不是friend的。

2.静态成员可以被继承，类的模型是所有对象的数据成员单独存储，但是所有成员函数和静态成员是共用一段空间的。
